# Pushup Bot 💪

Telegram-бот для командного 15-тижневого челенджу з віджимань.

Щодня о вказаний час бот надсилає тренування у групу. Учасники відмічають виконання через `/done`. Щоп'ятниці — автоматичний підсумок тижня.

## Що це таке

Pushup Bot — Telegram-бот для командного тренувального челенджу. Автоматично надсилає щоденні тренування у групу, відстежує виконання кожного учасника і підтримує командний дух через різноманітні фрази та святкові повідомлення.

Програма за замовчуванням: 15-тижнева прогресивна програма віджимань із 5 підходами щодня.

---

## Можливості

- Щоденні нагадування з тренуванням у вказаний час
- 15-тижнева прогресивна програма (підходи та кількість зростають щотижня)
- Відмітка виконання через `/done`
- Відстеження статусу команди в реальному часі
- Святкове повідомлення коли всі учасники виконали тренування
- Щоп'ятничний підсумок тижня з результатами кожного
- Мотиваційні фрази на вимогу (`/motivation`)
- Більше 490 унікальних фраз з антиповтором
- Різні повідомлення залежно від дня тижня (понеділок, п'ятниця, вихідні)
- Атомарний запис стану (state.json ніколи не пошкоджується)

## Встановлення

**Вимоги:** Python 3.13+

```bash
# Клонуй репозиторій
git clone <repo-url>
cd pushup-bot

# Створи та активуй віртуальне оточення
python -m venv .venv
source .venv/bin/activate       # macOS / Linux
# .venv\Scripts\activate        # Windows

# Встанови залежності
pip install -r requirements.txt

# Скопіюй .env.example та заповни
cp .env.example .env
```

Відредагуй `.env` — встав свій `BOT_TOKEN`, `CHAT_ID` та імена учасників.

Запуск:

```bash
python main.py
```

## Конфігурація

Всі налаштування у файлі `.env` (не комітити у git):

```env
BOT_TOKEN=your_bot_token_here
CHAT_ID=-1001234567890
PARTICIPANTS=Діма=@dlevinn,Толя=@anatoliireva,Юра=@lurook
REMINDER_TIME=12:00
REMINDER_TIMEZONE=Europe/Warsaw
FRIDAY_SUMMARY_TIME=18:00
PHRASE_HISTORY_SIZE=10
```

| Змінна | Обов'язкова | За замовчуванням | Опис |
|--------|-------------|-----------------|------|
| `BOT_TOKEN` | Так | — | Токен бота від @BotFather |
| `CHAT_ID` | Так | — | ID групи (від'ємне число для груп) |
| `PARTICIPANTS` | Так | — | Учасники у форматі `Ім'я=@username` через кому |
| `REMINDER_TIME` | Ні | `12:00` | Час щоденного нагадування (HH:MM) |
| `REMINDER_TIMEZONE` | Ні | `Europe/Warsaw` | IANA timezone |
| `FRIDAY_SUMMARY_TIME` | Ні | `18:00` | Час п'ятничного підсумку (HH:MM) |
| `PHRASE_HISTORY_SIZE` | Ні | `10` | Кількість фраз, що не повторюються |

**Як отримати CHAT_ID:**
1. Додай бота до групи.
2. Відправ будь-яке повідомлення у групу.
3. Відкрий `https://api.telegram.org/bot<TOKEN>/getUpdates` — знайди `chat.id`.
4. Для груп це від'ємне число (наприклад `-1001234567890`).

## Команди бота

| Команда | Опис |
|---------|------|
| `/today` | Показує тренування на сьогодні |
| `/done` | Відмічає виконання тренування |
| `/status` | Показує хто вже виконав сьогодні |
| `/week` | Показує поточний тиждень програми |
| `/motivation` | Надсилає мотиваційну фразу |
| `/help` | Довідка по командах |

## Деплой на Render

1. Запуш код на GitHub.
2. Зайди на [render.com](https://render.com) → **New → Background Worker**.
3. Вибери свій репозиторій.
4. Налаштування:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python main.py`
   - **Instance Type:** Free або вище
5. У розділі **Environment Variables** додай всі змінні з `.env`:
   - `BOT_TOKEN`
   - `CHAT_ID`
   - `PARTICIPANTS`
   - `REMINDER_TIME`
   - `REMINDER_TIMEZONE`
   - `FRIDAY_SUMMARY_TIME`
   - `PHRASE_HISTORY_SIZE`
6. Натисни **Deploy**.

> **Background Worker** (не Web Service) працює постійно і не засинає від неактивності. Саме тому бот реєструється як Background Worker.

> **Примітка про безкоштовний план:** Render безкоштовний план може вимикати процеси після певного часу. Для стабільної роботи рекомендується платний план або альтернативний хостинг (fly.io, VPS).

## Як оновити розклад тренувань

Розклад знаходиться у `programs/pushups/schedule.json`. Формат:

```json
{
  "display_name": "Прогресивні Віджимання",
  "total_weeks": 15,
  "rest": "60 секунд",
  "weeks": [
    { "week": 1, "sets": [10, 10, 8, 8, 10] },
    { "week": 2, "sets": [12, 12, 10, 10, 12] }
  ]
}
```

Кожен елемент у `weeks` — це один тиждень. `sets` — кількість повторень у кожному підході. Можна мати будь-яку кількість підходів.

Після зміни перезапусти бота.

## Як додати учасника

Відкрий `.env` і додай нового учасника через кому у форматі `Ім'я=@username`:

```env
PARTICIPANTS=Діма=@dlevinn,Толя=@anatoliireva,Юра=@lurook,Сашко=@sashahandle
```

- **Ім'я** — відображуване ім'я у повідомленнях бота (Діма, Толя, Юра)
- **@username** — точний Telegram username учасника (без урахування регістру)

Бот матчить `/done` виключно за Telegram username. Якщо username не збігається — бот покаже помилку з виявленим username.

Перезапусти бота. Нічого більше змінювати не потрібно.

## Як змінити час нагадування

У `.env` відредагуй:

```env
REMINDER_TIME=09:00
REMINDER_TIMEZONE=Europe/Kyiv
```

Підтримуються будь-які [IANA timezone](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones): `Europe/Warsaw`, `Europe/Kyiv`, `UTC` тощо.

Перезапусти бота.

## Вирішення проблем

**Бот не реагує на команди**
- Переконайся, що `BOT_TOKEN` правильний.
- Переконайся, що бот доданий до групи і має права надсилати повідомлення.
- Перевір логи: `python main.py` виводить всі події у stdout.

**"Не знайшов тебе у списку учасників"**
- Перевір `PARTICIPANTS` у `.env` — ім'я має точно (без урахування регістру) співпадати з `first_name` або `username` у Telegram.

**Нагадування не надсилаються**
- Переконайся, що `CHAT_ID` правильний (від'ємне число для груп).
- Перевір `REMINDER_TIME` та `REMINDER_TIMEZONE`.
- Бот має бути адміністратором або мати право надсилати повідомлення у групу.

**Помилка "JobQueue недоступний"**
- Встанови залежність: `pip install 'python-telegram-bot[job-queue]'`
- Або переінсталюй: `pip install -r requirements.txt`

**П'ятничний підсумок не надсилається**
- Переконайся, що `FRIDAY_SUMMARY_TIME` встановлено у `.env`.
- Підсумок надсилається тільки у п'ятницю. Бот перевіряє день тижня перед відправкою.

**state.json пошкоджений**
- Видали файл `data/state.json` — бот створить новий з початковим станом.
- При цьому скинеться поточний тиждень до 1.
